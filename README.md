# Parakot

Монорепозиторий проекта `parakot.ru`.

## Структура

```text
landing/  - публичный лендинг
admin/    - будущая SPA-админка
api/      - будущий PHP REST API
docs/     - ТЗ, архитектура и заметки по проекту
```

## Почему один репозиторий

- лендинг, админка и API относятся к одному продукту
- удобно версионировать совместимые изменения вместе
- проще вести задачи, документацию и деплой
- легче не потерять синхронность между схемой данных и интерфейсами

## Текущий статус

- `landing/` содержит текущий HTML/CSS/JS прототип сайта
- `admin/` пока оставлен как заготовка под React SPA
- `api/` пока оставлен как заготовка под PHP REST backend
- `docs/` хранит текущий бриф и архитектурные решения

## Ближайший план

1. Зафиксировать архитектуру MVP админки и API.
2. Подготовить схему базы данных.
3. Собрать каркас `admin/` и `api/`.
4. Научить лендинг читать контент не из жесткой верстки, а из управляемых данных.

## Локальная работа

Пока репозиторий ведется локально. Когда каркас станет устойчивым, можно будет:

1. создать приватный репозиторий на GitHub
2. добавить `origin`
3. запушить текущую историю
